redis数据库基本操作

redis
redis
发布日期:   2020-07-24

redis下载安装

依次执行以下命令：

$ wget http://download.redis.io/releases/redis-3.2.8.tar.gz
$ tar -zxvf redis-3.2.8.tar.gz
$ sudo mv ./redis-3.2.8 /usr/local/redis
$ cd /usr/local/redis
$ sudo make
$ sudo make test
$ sudo make install

安装完之后，查看 /usr/local/bin，如果有如下即安装成功。

-rwxr-xr-x 1 root root 3344600 7月  24 14:38 redis-benchmark  # redis性能测试工具
-rwxr-xr-x 1 root root   35872 7月  24 14:38 redis-check-aof  # AOF文件修复工具
-rwxr-xr-x 1 root root 6771608 7月  24 14:38 redis-check-rdb  # RDB文件修复工具
-rwxr-xr-x 1 root root 3564272 7月  24 14:38 redis-cli  # redis命令行客户端
lrwxrwxrwx 1 root root      12 7月  24 14:38 redis-sentinel -> redis-server
-rwxr-xr-x 1 root root 6771608 7月  24 14:38 redis-server  # redis服务器

redis配置

安装完成之后，会在 redis 文件夹爱中产生一个 redis.conf 配置文件，把它移到 /etc/redis/ 中,修改其中内容进行配置。

 $ sudo cp redis.conf /etc/redis

配置文件中需要注意的如下（修改过后）：

bind 127.0.0.1                             # 默认绑定ip  
port 6379                                  # 默认端口
daemonize yes                              # 是否为守护进程（默认为no）
logfile "/var/log/redis/redis-server.log"  # 日志文件（需要新建）
databases 16                               # 默认16个数据库，编号0-15
dbfilename dump.rdb                        # 数据文件
dir /var/lib/redis                         # 数据文件存储路径（需要新建）
slaveof                                    # 主从配置

redis服务端和客户端的启动

服务端命令为 redis-server,redis-server --help 可查看帮助文档。
启动与关闭服务器如下：

$ sudo redis-server /etc/redis/redis.conf  # 指定加载的配置文件，启动服务器
$ ps aux | grep redis                      # 查看进程pid
$ sudo kill -9 pid                         # 杀死服务器，即关闭

客户端命令为 redis-cli,redis-cli --help 可查看帮助文档。
直接输入 redis-cli 即可连接服务端，输入 ping 起到检查连接的作用。

$ redis-cli
127.0.0.1:6379> ping
PONG

利用 select 命令切换制定数据库，如：

select 5

redis数据操作

redis命令大全 很多命令可以去查找

以下代码有公式形式，匹配例子。

string类型

设置的健不存在则添加，存在则修改。

  • 设置键值
      set key value
  set name itcast
  • 设置键值及过期时间，以秒为单位
      setex key seconds value
  setex aa 3 aa
  • 设置多个键值
      mset key1 value1 key2 value2 ...
  mset a1 python a2 java a3 c
  • 追加值
      append key value
  append a1 haha
  • 根据键获取值，如果不存在则返回 nil 
      get key
  get name
  • 获取多个键的值
      mget key1 key2 ...
  mget a1 a2 a3

    键命令

  • 查找键，参数支持正则表达式
      keys pattern
  keys *       # 查看所有键
  keys ‘a*’    # 包含 a 的键
  • 判断键是否存在，存在返回1，否则返回0
      exists key1
  exists a1
  • 查看对应的值的类型
      type key
  type a1
  • 删除键及对应的值
      del key1 key2 ... 
  del a2 a3
  • 设置过期时间，以秒为单位
      expire key seconds
  expire 'a1' 3
  • 查看有效时间
      ttl key
  setex bb 10 bb
  ttl bb

hash类型

hash 用于存储对象，对象的结构为属性，值。值的类型为string。

  • 设置单个属性
      hset key field value
  hset user name itheima
  # 如果报错，则表示为数据库快照，可执行下面命令使数据持久化
  config set stop-writes-on-bgsave-error no
  • 设置多个属性
      set key field1 value1 field2 value2
  set u2 name itcast age 11
  • 获取指定键（对象）所有的属性
      hkeys key
  hkeys u2
  • 获取一个属性的值
      hget key field
  hget u2 name
  • 获取多个属性的值
      hmget key field1 field2
  hmget u2 name age
  • 获取所有属性的值
      hvals key
  hvals u2
  • 删除整个hash键及值，使用 del 命令
  • 删除属性，属性对应的值会被一起删除
      hdel key field1 field2 ...
  hdel u2 age

list类型

列表的元素类型为 string，按照插入顺序排序。

  • 在左侧插入数据
      lpush key value1 value2 ...
  lpush a1 a b c
  • 在右侧插入数据
      rpush key value1 value2 ...
  rpush a1 0 1
  • 在指定元素的前或后插入新元素
      linsert key before或after 现有元素 新元素
  linsert a1 before b 3
  • 获取列表里制定范围内的元素
      lrange key start stop
  lrange a1 0 -1
  • 设置指定索引位置的元素值
      lset key index value
  lset a 1 z
  • 删除指定元素
    • 将列表中前 count 次出现的值为 value 的元素移除
    • count > 0; 从头往后移除
    • count < 0; 从后往头移除
    • count = 0; 全移除
      irem key count value

set类型

无序集合，元素为 string 类型， 唯一性，不重复。对于集合没有修改操作。

  • 添加元素
      sadd key member1 member2 ...
  sadd a3 zhangsan sili wangwu
  • 返回所有元素
      smembers key
  smember a3
  • 删除指定元素
      srem key memebr1
  srem a3 wangwu

zset类型

有序集合，元素为 string 类型， 唯一性，不重复。每个元素都会关联一个double类型的score，表示权重。通过权重将元素从小到大排序，和 set 一样没有修改操作。

  • 添加元素
      zadd key score1 member1 score2 member2 ...
  sadd a4 4 lisi 5 wangwu 6 zhaoliu 3 zhangsan
  • 返回指定范围内元素
      zrange keyy start stop
  zrange a4 0 -1
  • 返回 score 值在 min 和 max 之间的元素（闭区间）
      zrangebyscore key min max
  zrangebyscore a4 5 6
  • 查看某 member 的 score 值
      zscore key member
  zscore a4 zhangsan
  • 萨和年初指定元素
      zrem key member1 member2 ...
  zrem a4 zhangsan
  • 删除 score 值在 min 和 max 之间的元素（闭区间）
    • 将列表中前 count 次出现的值为 value 的元素移除
    • count > 0; 从头往后移除
    • count < 0; 从后往头移除
    • count = 0; 全移除
      zremrangebyscore key min max
zremrangebyscore a4 5 6

sr = StrictRedis(host = 'localhost', port = 6379, db = 0)
# 上述都为默认值，可简写
sr = StrictRedis()

根据不同的类型，拥有不同的实例方法可以调用，与前面的 redis 命令对应，方法的参数与命令的参数也一致。

查看rdb文件

redis存储下来的数据文件是 dump.rdb，用vim打开是一堆乱码，在个github上发现了，查看rdb文件的工具。https://github.com/sripathikrishnan/redis-rdb-tools 执行以下命令，可以以json形式查看数据。

rdb -c json dump.rdb

由于rdb是一种快照备份，在客户端修改数据过后，不会立即在dump.rdb中显示出来，若想立刻看到，需要在客户端执行BGSAVE异步后台保存，再查看就可以了。注：conf文件中有快照的配置，如下：

# 如果不备份就将 下边3个参数注释掉
# 刷新快照到硬盘，必须满足两者要求才会触发
save 900 1     # 即900秒之后至少1个关键字发生变化
save 300 10    # 必须是300秒之后至少10个关键字发生变化
save 60 10000  # 必须是60秒之后10000个关键字发生变化
