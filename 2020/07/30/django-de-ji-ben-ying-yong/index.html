django的基本应用

django
发布日期:   2020-07-30

项目与应用创建

django-admin startproject test1  # 创建 test1 项目
python37 manage.py startapp booktest  # 创建 booktest 应用
  • 项目目录结构
    1. __init__.py 说明是一个 python 包；
    2. settings.py 项目的配置文件；
    3. urls.py 进行 url 路由的配置；
    4. wsgi.py web 服务器和 Django 交互的入口
    5. manage.py 项目的管理文件。
  • 应用目录结构
    1. __init__.py 说明是一个 python 包；
    2. model.py 写和数据库相关的功能；
    3. views.py 接受请求，进行处理，与 M 和 T 进行交互，返回应答，定义处理函数，称为视图函数；
    4. test.py 写测试代码的文件；
    5. admin.py 网站后台管理相关的文件

还需要建立应用与项目之间的联系，需要对应用进行注册。
修改settings.py中的 INSTALLED_APPS 配置项，加入自己的应用名称即可。

INSTALLED_APPS = (
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'booktest',  # 注册应用
)

配置 mysql 数据库

Django 自带的数据库是 sqlite 相对小型，在大项目中用的更多的是 oracle 和 mysql，所以要替换。

模型是对数据库的相关操作，利用 Django 配置 mysql 数据库，需要修改settings.py中的 DATABASE。

DATABASES = {
    'default': {
        # 'ENGINE': 'django.db.backends.sqlite3',
        'ENGINE': 'django.db.backends.mysql',  
        # 'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
        'NAME': 'qay',            # 数据库名称
        'USER': 'root',           # 登陆数据库的用户名
        'PASSWORD': 'qin200053',  # 登陆密码
        'HOST': 'localhost',      # 登陆地址
        'PORT': '3306',           # 端口号，mysql默认端口号是3306
    }
}

注意：django不会生成数据库，需要我们自己创建。

我的 Ubuntu 版本是20.04，mysql官网上只有8.0才支持20.04，于是我直接安装的 mariadb 暂时作为 mysql 的替代品，目前看来命令没什么区别，希望以后不会出什么bug。

sudo apt-get mariadb-server mariadb-client  # 安装
  • 开启关闭服务，登陆退出。
      sercive mysql start
  service mysql stop
  sudo mysql -uroot
  • 避免使用 sudo 登陆,需要执行sudo vim /etc/mysql/my.cnf，然后添加如下语句，再重启服务即可。
      [mysqld]
  skip-grant-tables
  • 修改mysql密码。
      mysql -uroot -p
  use mysql 
  update user set authentication_string=password('xxxxxx') where User='root';
  • 创建新的数据库
      mysql -uroot -p
  show databases;
  create database qay charset=utf8;
  show databases;

处理完 mysql，在启动项目之前，需要sudo pip37 install pymysql，并在项目的__init__.py中加入如下语句，就可以了。

import pymysql
pymysql.install_as_MySQLdb()

模型类

我们需要设计和数据库中表相对应的类，称为模型类，本质上就是在models.py中建立 class，其必须继承于 models.Model类。

class BookInfo(models.Model):
    '''图书模型类'''
    # 图书名称，字符串，最长20
    btitle = models.CharField(max_length=20)
    # 出版日期，日期类型
    bupdate = models.DateField()

设计完模型类之后，将其生成为数据库中的表：

  • 生成迁移文件python37 manage.py makemigrations，文件生成在 migrations 文件夹中；

  • 执行迁移生成表python37 manage.py migrate，这样去数据库里就能查看到了，创建的表名即为应用名_类名。

  • 接下来就可以通过python语句，对数据进行操作了，利用 python37 manage.py shell进行操作：

      >>> from booktest.models import BookInfo
  >>> b = BookInfo()
  >>> b.btitle = '三体'
  >>> from datetime import date
  >>> b.bupdate = date(1990,1,1)
  >>> b.save()  # 通过 save 进行插入和修改
  >>> b_all = BookInfo.objects.all()   # 拿到所有实例，即表中所有条目，返回一个列表
  >>> b2 = BookInfo.objects.get(id=1)  # 拿到实例，可以借此查询
  >>> b_cond = BookInfo.objects.get(btitle='三体')  # 按条件查询
  >>> type(b2)
  <class 'booktest.models.BookInfo'>
  >>> b2.btitle
  '三体'
  >>> b2.bupdate
  datetime.date(1990,1,1)
  >>> b2.delete()  # 通过 delete 删除
class HeroInfo(models.Model):
    hname = models.CharField(max_length=20)
    hgender = models.BooleanField(default=False)
    hcomment = models.CharField(max_length=128)
    # 关系属性 hbook，建立图书类和英雄人物类之间的一对多关系
    hbook = models.ForeignKey('BookInfo')

表之间存在关系。

  • Models.ForeignKey 可以建立两个模型之间一对多的关系，django 在生成表的时候，就会在多的表中创建一列作为外键，建立两个表之间一对多的关系。关系属性对应的表的字段名格式为关系属性名_id。

    以上面的类作为例子，很多英雄可能都属于一本书，所以在‘多类’中添加关系属性，赋值时要赋‘一类’的实例，在实际的表中，h的属性hbook_id的值即为b的主键id值：

      b = BookInfo()
  h = HeroInfo()
  h.hbook = b

  • 建立了关系之后，可以根据关系查询：

    由一查多：b.heroinfo_set.all()
    由多查一：h.hbook

视图

在 Django 中，通过浏览器去请求一个页面时，使用视图函数来处理这个请求，视图函数处理之后，要给浏览器返回页面内容。

  • 在views.py中定义视图函数

    视图函数必须有一个参数request，处理之后需要返回一个 HttpResponse 的类对象。

      # http://127.0.0.1:8000/index
  def index(request):
      return HttpResponse("nb!")

  • 配置项目路由

      urlpatterns = [
      url(r'^admin/', include(admin.site.urls)),
      url(r'^', include('booktest.urls')),
  ]

  • 配置应用路由，需要自己在应用目录下创建urls.py

      urlpatterns = [
      url(r'index$', views.index),
  ]

  • 路由匹配过程

    当浏览器中输入http://127.0.0.1:8000/index，会自动除去前面的http://127.0.0.1:8000/用剩下的部分进行匹配，这里即index，先根据项目路由中的urlpatterns从上到下匹配，匹配成功后，根据后面的include()确定下一部去哪个应用进行匹配，这里即booktest.urls，然后再到urlpatterns从上到下匹配，直到匹配成功为止。

    匹配时严格根据正则规则匹配。（在新版本的django中函数 url 改成了函数 path，已经不需要正则表达式了）

模板

  • 创建模板文件夹templates,并在settings.py的 TEMPLATES 中配置模板目录，通常可能有很多应用，所以在templates再以应用名建立文件夹booktest
      'DIRS': [os.path.join(BASE_DIR, 'templates')],
  • 使用模板文件分为三步：
    1. 加载模板文件：去模板目录下获取html文件内容，得到一个模板对象；
    2. 定义模板上下文：像模板文件传递数据，传递时以键值对形式传递；
    3. 模板渲染：得到一个标准的html内容。
      def my_render(request, template_path, context_dict):
 temp = loader.get_template('booktest/index.html')
 context = RequestContext(request, {})  # 这里传了空字典
 res_html = temp.render(context)
 return HttpResponse(res_html)
    4. 以上三步可以用一个函数render完成:
      def index(request):
 # 传了两个变量，一个字符串一个列表
 return render(request, 'booktest/index.html',
           {'content':'hello world','list':list(range(1,10))})
  • 模板文件不仅仅时一个html文件，可以说有自己的语法，变量需要用两组花括号包裹，循环遍历或判断要用花括号和百分号包裹，下面给出一个例子：
      <!DOCTYPE html>
  <html lang="en">
  <head>
      <meta charset="UTF-8">
      <title>mplate file</title>
  </head>
  <body>
      <h1>this is a template file</h1>
      use the template variable: <br>
      {{ content }} <br>
      use the list: <br>
      {{ list }} <br>
      for loop: <br>
      <ul>
          {% for i in list %}
                <li>{{ i }}</li>
            {% endfor %}
      </ul>
  </body>
  </html>

很简单的一个实例

做一个很简单的实例，输入 http://127.0.0.1:8000/books 进入页面显示书名，点击书名能跳转到书所包含的人物信息。

  • 最重要的就是设计视图函数

      def show_books(request):
      books = BookInfo.objects.all()
      return render(request, 'booktest/show_books.html',
                  {'books':books})

  def detail(request, bid):
      book = BookInfo.objects.get(id=bid)
      heros = book.heroinfo_set.all()
      return render(request, 'booktest/detail.html',
          {'book':book, 'heros':heros})

  • 配置各个页面的路由

      urlpatterns = [
      url(r'^books$', views.show_books),
      url(r'^books/(\d+)$', views.detail),
  ]

  • 编写模板呈现页面

          # show_books.html
      Information of Book is Followed:
      <ul>
          {% for book in books %}
            <li><a href="/books/{{ book.id }}">{{ book.btitle }}</a></li>
            {% endfor %}
      </ul>

      # detail.html
      <h1>{{ book.btitle }}</h1>
      Information of Heros is Followed:<br>
      <ul>
          {% for hero in heros %}
                <li>{{ hero.hname }}--{{ hero.hcomment }}</li>
            {% empty %}
                <li>no Information of Hero</li>
            {% endfor %}
      </ul>
