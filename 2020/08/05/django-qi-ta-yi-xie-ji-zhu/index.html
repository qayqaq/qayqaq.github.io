静态文件

网页里使用的 css, js, images属于于静态文件。

  • 新建静态文件夹static。
  • 配置静态文件所在的物理目录。settings.py
    • STATIC_URL设置访问静态文件对应的url；
    • STATICFILES_DIRS设置静态文件所在的物理目录
  • 动态生成静态文件路径
      <!DOCTYPE html>
  {% load staticfiles %}  # 加载静态文件
  <html lang="en">
      <img src="{% static 'images/123.jpg' %}">  # 动态拼接静态文件路径
  </html>
  • 加载目录：先去配置的statics目录找，没找到再去各个应用下的statics目录找。

中间件

中间件函数是django框架给我们预留的函数接口，让我们可以干预请求和应答的过程。

中间件函数会在每一个视图函数执行前执行，所以我们可以修改或定义中间件函数来达到某向目的，比如：禁止某个 ip 地址访问所有网页，这个功能是可以通过装饰器实现的，但是需要在每个视图函数前面都加上装饰，很麻烦，而用中间件函数则很方便。

EXCLUDE_IPS = []
def blocked_ips(view_finc):
    def wrapper(request, *view_args, **view_kwargs):
        user_ip = request.META['REMOTE_ADDR']
        if user_ip in EXCLUDE_IPS:
            return HttpResponse('<h1>Forbidden<h1>')
        else:
            return view_finc(request, *view_args, **view_kwargs)
    return wrapper

@blocked_ips
def static_test(request):
    print(settings.STATICFILES_FINDERS)
    # 'django.contrib.staticfiles.finders.FileSystemFinder', \
    # 'django.contrib.staticfiles.finders.AppDirectoriesFinder'
    return render(request, 'booktest/static_test.html')

@blocked_ips
def index(request):
    return render(request, 'booktest/index.html')

  • 获取浏览器端的 ip 地址
    使用 request 对象的 META 属性： request.META['REMOTE_ADDR']

  • 在应用中新建middleware.py

  • 定义中间件类，在类中定义中间件预留函数，包括：

    • __init__: 服务器响应第一个请求的时候调用；

    • process_request: 是在产生 request 对象，进行 url 匹配之前调用；

    • process_view: 是 url 匹配之后，处理视图函数之前；

    • process_response: 视图函数调用之后，内同返回给浏览器之前；

    • process_exception: 视图函数出现异常，会调用这个函数；

      以上叙述即是正常工作流程：浏览器向django第一次发送请求，django产生一个request同时执行__init__，之后执行process_request，之后进行url匹配，之后执行process_view，之后调用视图函数，之后执行process_response，之后给出相应，在过程中若视图函数异常，执行process_exception。

      如果注册的多个中间件类中包含process_exception时，调用的顺序根注册的顺序是相反的。

      # middleware.py
class BlockedIPSMiddleWare(object):
  EXCLUDE_IPS = []
  def process_view(self, request, view_func, *view_args, **veiw_kwargs):
      user_ip = request.META['REMOTE_ADDR']
      if user_ip in BlockedIPSMiddleWare.EXCLUDE_IPS:
          return HttpResponse('<h1>Forbidden<h1>')

class TestMiddleWare(object):
  def __init__(self):
      print('----init----')
  def process_request(self, request):
      print('----process_request----')
  def process_view(self, request, view_func, *view_args, **view_kwargs):
      print('----process_view----')
  def process_response(self, request, response):
      print('----process_reponse----')
      return response

class ExceptionTest1MiddleWare(object):
  def process_exception(self, request, exception):
      print('----process_exception1----')
      print(exception)

class ExceptionTest2MiddleWare(object):
  def process_exception(self, request, exception):
      print('----process_exception2----')

  • 注册中间件类：结合前面所说的顺序问题，当视图函数发生异常时，先执行ExceptionTest2MiddleWare。

      MIDDLEWARE_CLASSES = (
      'booktest.middleware.BlockedIPSMiddleWare',
      'booktest.middleware.TestMiddleWare',
      'booktest.middleware.ExceptionTest1MiddleWare',
      'booktest.middleware.ExceptionTest2MiddleWare',
  )

  • 还有一点需要注意的是，如果在流程中的任意位置返回了response，则直接跳到整个流程的最后一步，执行process_response，之后给出相应，这也是为什么最开始的装饰器可以达到干预应答的效果的原因。

# models.py
class AreaInfo(models.Model):
    asize = models.BooleanField(default=False)
    atitle = models.CharField(verbose_name='标题', max_length=20)
    aParent = models.ForeignKey('self', null=True, blank=True)

    def __str__(self):
        return self.atitle

    def title(self):
        return self.atitle
    title.admin_order_field = 'atitle'
    title.short_description = '地区名称'

    def parent(self):
        if self.aParent is None:
            return ''
        return self.aParent.atitle
    parent.short_description = '父级地区名称'

  • 配置上传文件保存目录

    • 新建media文件夹
    • MEDIA_ROOT = os.path.join(BASE_DIR, 'static/media')

  • 后台管理页面上传图片

    • 设计模型类
        class PicTest(models.Model):
      goods_pic = models.ImageField(upload_to='booktest')
    • 迁移生成表格
    • 注册模型类

  • 用户自定义页面上传图片

    • 定义用户上传图片的页面并显示，是一个自定义的表单

        <form method="post" enctype="multipart/form-data" action="/upload_handle">
      {% csrf_token %}
      <input type="file" name="pic"><br>
      <input type="submit" value="上传">
  </form>

    • 定义接受上传文件的视图函数：request 对象有一个FILES属性，类似于字典，通过request.FILES可以获取上传文件的处理对象。当上传文件不大于2.5M，文件放在内存中，若大于2.5M，文件内容写入临时文件中。

        def show_upload(request):
      return render(request, 'booktest/upload_pic.html')

  def upload_handle(request):
      # 1. 获取上传文件的处理对象
      pic = request.FILES['pic']
      print(type(pic))
      # print(pic.name)
      # pic.chunks()
      # 创建一个文件
      save_path = '%/booktest/%s'%(settings.MEDIA_ROOT, pic.name)
      with open(save_path, 'wb') as f:
          # 获取上传文件的内容 并写到创建的文件中
          for content in pic.chunks():
              f.write(content)
      # 在数据库中保存上传记录
      PicTest.objects.create(goods_pic='booktest/%s'%pic.name)
      # 返回
      return HttpResponse('ok')
